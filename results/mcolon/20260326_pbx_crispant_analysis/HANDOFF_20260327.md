# PBX Crispant Analysis Handoff

Date: 2026-03-27

## Summary

This analysis pass focused on embryo-level pairwise classification and misclassification structure for the PBX crispant dataset built from `20260304` and `20260306`.

The main goal was to understand:

1. whether `pbx4_crispant` contains a wildtype-like subclass relative to `inj_ctrl`
2. whether `pbx4_crispant` and `pbx1b_pbx4_crispant` behave like a phenotype continuum
3. whether the `pbx1b_pbx4_crispant` phenotype looks like an enhancement of `pbx4_crispant`

The work produced a set of pairwise signed-margin plots, morphology follow-up scripts, embryo subclass labels, and an initial all-pairs overlap analysis.

## Main Findings So Far

### Negative control

`inj_ctrl vs wik_ab` is weak, as expected for a negative control.

From [pairwise_auroc_summary_pbx_controls_embedding_all_pairs.csv](/net/trapnell/vol1/home/mdcolon/proj/morphseq/results/mcolon/20260326_pbx_crispant_analysis/results/misclassification/embedding/pairwise_auroc_summary_pbx_controls_embedding_all_pairs.csv):

- `inj_ctrl_vs_wik_ab`: mean AUROC `0.513`

### Stronger separations

The strongest pairwise separations are:

- `inj_ctrl vs pbx1b_pbx4_crispant`
- `wik_ab vs pbx1b_pbx4_crispant`
- `wik_ab vs pbx4_crispant`
- `inj_ctrl vs pbx4_crispant`

The hardest PBX pair is still:

- `pbx4_crispant vs pbx1b_pbx4_crispant`

with mean AUROC around `0.524`.

### PBX4 enhancement-style overlap result

The key PBX4 anchor test was:

- set A: `inj_ctrl-like/pbx4_true`
- set B: `pbx1b+4-like/pbx4_true`

From [summary_pbx_controls_embedding_all_pairs.txt](/net/trapnell/vol1/home/mdcolon/proj/morphseq/results/mcolon/20260326_pbx_crispant_analysis/results/misclassification/embedding/summary_pbx_controls_embedding_all_pairs.txt):

- observed overlap: `2`
- expected overlap: `8.571`
- overlap over expected: `0.233`
- `q_depletion = 0.000359`

This is in the expected depletion direction for the enhancement idea.

Related interpretation from the pair-specific overlap view:

- among PBX4 embryos that are `inj_ctrl-like` and `wik_ab-like`, there is effectively no meaningful overlap with the `pbx1b+4-like` PBX4 subset
- this supports the idea that the `pbx1b+4` comparison is shifting the PBX4 phenotype distribution away from the wildtype side rather than simply broadening noise around wildtype
- stated differently, the PBX4 embryos that still look wildtype/control-like do not appear to be the same embryos that look `pbx1b+4-like`

## What Is Still Unclear

The analysis is not yet presentation-ready.

Main reasons:

- the PBX4 overlap heatmap required several iterations and the right abstraction is still not fully settled
- there is still ambiguity between pair-specific labels and grouped label families
- the current overlap summary is statistically useful but not yet visually obvious
- some of the labels and figures are still exploratory rather than final communication figures

In short, the current state is analytically productive but narratively messy.

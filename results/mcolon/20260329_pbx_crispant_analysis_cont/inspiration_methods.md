Method	Core object	How it avoids failure / collapse	What to borrow	Main limitation for your use
UMAP	Low-dimensional embedding of a fuzzy kNN graph	Uses local neighbor-based attraction, explicit repulsion, stable initialization, and a fixed annealed optimization budget rather than all-pairs condensation.  ￼	kNN-local attraction, explicit repulsion, practical fixed-epoch optimization	Does not directly model time-coupled trajectories or extract branching skeletons.  ￼

Diffusion condensation / Multiscale PHATE	Iterative coarse-graining of data geometry across scales	Lets points move together gradually, but the useful output is often the intermediate metastable scales, not the final fully condensed state.  ￼	Save intermediate states, analyze persistence across iterations/scales, do not over-trust the endpoint	Does not by itself give you an explicit branch scaffold or trajectory elasticity.  ￼
Elastic Principal Graphs (ElPiGraph)	Branching graph scaffold fit to data	Avoids free-particle blob collapse by fitting an elastic graph with topology control, smoothness penalties, and penalties for excessive branching.  ￼	Explicit branch scaffold, topology control, post hoc branch extraction	Not designed around your persistence-gated time-coupled interaction dynamics.  ￼
Force-directed graph layout	Positions of nodes in a graph	Uses attractive forces to keep related nodes together and repulsive forces to keep nodes separated, which is the classic anti-collapse design pattern.  ￼	Good mental model for balancing attraction vs repulsion	Graph layout itself is not a developmental trajectory model.  ￼

Use UMAP-style locality + repulsion to fix the immediate slice-collapse pathology, borrow diffusion-condensation thinking so intermediate scales matter, and use ElPiGraph-style scaffolds later if you want an explicit branching object rather than hoping the particle cloud becomes a tree by itself.  ￼

## Target module structure

```
src/data_pipeline/analysis_ready/
  core/
    assemble.py                    ← promote join logic from existing stub; no I/O
  entrypoints/
    assemble_analysis_ready.py     ← CLI: parse args → load → call core → writers.py
  io/
    loaders.py                     ← load consolidated_snip_features.csv + qc_flags.csv
    writers.py                     ← validate → write analysis_ready.csv + schema.json → write sentinel
  validators.py                    ← validate_analysis_ready(df), assert_unique_snip_id(df), assert_1to1_join(left, right)
```

---

## Join semantics — hard constraints

The join is a **validated 1:1 join on `snip_id`**. These are hard errors:
- Duplicate `snip_id` on either input side
- `snip_id` present in one input but not the other
- Row count of output ≠ row count of each input
- Column name collision between inputs (except `snip_id` itself)

Implement these checks in `validators.py::assert_1to1_join(left_df, right_df)` and call
it before finalizing the output.

**`analysis_ready.schema.json`:** Required MVP output. Generated by `io/writers.py` from
the canonical schema definition at write time. Not hand-authored.

**`embedding_calculated`:** Must be present, non-null boolean, all `False` for MVP.

## Snakemake rule to add

Add one rule `assemble_analysis_ready` in:
`src/data_pipeline/pipeline_orchestrator/Snakefile`

Place it after the `consolidate_qc` rule (Agent A's last rule).

```python
rule assemble_analysis_ready:
    input:
        features = EXPERIMENT_METADATA_DIR / "{experiment}" / ... / "consolidated_snip_features.csv",
        features_validated = ...,
        qc_flags = QUALITY_CONTROL_DIR / "{experiment}" / "consolidated" / "qc_flags.csv",
        qc_validated = QUALITY_CONTROL_DIR / "{experiment}" / "consolidated" / ".qc_flags.validated",
    output:
        table = ANALYSIS_READY_DIR / "{experiment}" / "analysis_ready.csv",
        validated = ANALYSIS_READY_DIR / "{experiment}" / ".analysis_ready.validated",
        schema_json = ANALYSIS_READY_DIR / "{experiment}" / "analysis_ready.schema.json",
    shell:
        """
        "$PYTHON" -m data_pipeline.analysis_ready.entrypoints.assemble_analysis_ready \
          --features-csv "{input.features}" \
          --qc-flags-csv "{input.qc_flags}" \
          --output-csv "{output.table}" \
          --output-schema-json "{output.schema_json}"
        """
```

**Also add** `ANALYSIS_READY_DIR` path constant near the other directory constants at the
top of the Snakefile:
```python
ANALYSIS_READY_DIR = DATA_ROOT / "analysis_ready"
```
